BWC triangle — exploration

Working through the fixed-hue black / white / color triangle as a mental model, a naming scheme, a general color picker, and a constrained picker for our tokens.

1.The math

Pick a hue and freeze it. Every color at that hue is a mix of white, black, and the pure hue. Call those three primaries W, B, C (for whiteness, blackness, colorfulness). Any color is a weighted sum: 

color = W · white + B · black + C · pure_hue
W + B + C = 1   (normalized)

Three weights summing to 1 means only two are independent. That's why a triangle (a 2D simplex) is the right picture — points inside it address every possible mix by their barycentric coordinates (w, b, c).

This is Alvy Ray Smith's HWB model (1996), reachable from HSV by: 

W = (1 - S) · V
B = 1 - V
C = 1 - W - B = S · V

CSS exposes it as hwb(h w% b%). The browser does the conversion for us when we render.

Barycentric coordinates

Pick three triangle vertices V_W, V_B, V_C. Any point P = (x, y) inside has unique (w, b, c) with w + b + c = 1 satisfying P = w · V_W + b · V_B + c · V_C. Solving gives a small linear system (one determinant, two ratios). Go look at barycentric() in the script.

sRGB vs perceptual

CSS hwb() is sRGB-based (derived from HSV). It's geometrically elegant but not perceptually uniform — equal steps in W don't look equally different. Our current system uses OKLCH, which is perceptually uniform but unbounded in C (real displays clip).

For this exploration we use CSS hwb() directly so the math and the picture line up. If we commit to this model for tokens we'd likely still store values in OKLCH for perceptual cadence and just name them in W/B/C terms.

2.As a naming convention

If every mark is (scale, w, b) (with c = 1 - w - b implied), tokens name themselves after their position on the triangle. Grid-style — pick a resolution, quantize, and you get a finite palette.

Proposal A — round W/B to tenths

{scale}-w{W×10}-b{B×10}

surface-w9-b0   paper   (near-pure white, zero black, zero color)
surface-w0-b9   ink     (pure ink)
surface-w0-b3   mid-grey
accent-w0-b2    deep brand
accent-w0-b0    pure accent at that hue
accent-w3-b0    tinted accent (washed toward white)

Readable, names are addresses. The bounded grid (11×11 with w + b ≤ 10) gives 66 legal cells, which is larger than we probably need.

Proposal B — named anchors on a rough grid

{scale}-bg     =  w9,b0   (theme flips to w0,b9)
{scale}-bg+1   =  w8,b0
{scale}-ink    =  w0,b9   (theme flips to w9,b0)
{scale}-ink-1  =  w0,b8
{scale}-tint-3 =  w6,b0   (mostly white with some color)
{scale}-full   =  w0,b0   (pure hue)

Fewer, meaningful names. Less flexible per-mark but much easier to adopt without a dictionary.

Where contrast pairs live geometrically

  • White on black — W corner on B corner.
  • Black on white — B corner on W corner.
  • Color on white — C corner on W corner.
  • Color on black — C corner on B corner.
  • White on color — W corner on C corner.
  • Black on color — B corner on C corner.

Every classical contrast pair is literally "one corner on another." The empty center is where UI mostly lives — dialled-down tints, subtle borders, calmer accent text.

3b.Perceptually uniform picker

Same triangle, same hue ring, same (W, B, C) ternary — but interpolation happens in OKLCH instead of HWB. Each interior point is L = w·1 + b·0 + c·L* and C = c·C* where (L*, C*) is a display-safe peak for the pure hue. The final pixel is oklch(L, C, hue).

Visually the midtones don't collapse toward black the way sRGB HWB does. Equal barycentric distance now corresponds to equal perceived distance, which is the property we actually want when reasoning about contrast.

4.Constrained picker — our tokens

The triangle is pinned to each scale's hue. Dots mark where our 12 tokens land. The cluster along the W–B edge (zero C) is the "mostly grey" side; deviations toward the C corner show how colorful each token actually is.

Surface scale — h 162°, c 0.010
Accent scale — h 162°, c 0.070

Observations

  • The surface scale lives almost entirely on the W–B edge (C ≈ 0). The tiny chroma moves so little that the dots read as a straight line. That's "greyscale" by construction.
  • The accent scale bows out toward C, peaking around step 8–9. Tokens at the extremes (1, 12) pull back toward the W–B edge because of the per-step chroma taper.
  • The empty triangle interior on the accent side is the unexplored design space — marks that are the same lightness as today's tokens but meaningfully more or less colorful.

5.Quantizing the triangle

A 1D scale is just a ruler divided into N marked points. The triangle is a 2-simplex — two degrees of freedom — so the natural analogue is a ternary grid. At resolution N, every token is a triple (i, j, k) of non-negative integers with i + j + k = N, interpreted as barycentric weights (W, B, C) = (i/N, j/N, k/N).

The grid has (N+1)(N+2)/2 cells. A few sizes: 

N = 3 →  10 tokens per hue
N = 4 →  15
N = 5 →  21
N = 6 →  28
N = 8 →  45

Color math runs in OKLCH for perceptually-uniform lightness. The triangle's C vertex is fixed at L* = 0.55, and for each hue we use the max in-gamut C* at that L*. Consequence: the same address (i, j, k) resolves to the same L regardless of hue — only the chroma changes across hues. That's the cross-hue cadence you want for a design-system palette.

Naming

Two readable schemes on top of the ternary address: 

{scale}-w{i}b{j}        e.g.  surface-w4b0  (pure white tint)
                             accent-w0b0   (pure hue)
                             accent-w2b2   (dusty mid-tone)

{scale}-N{k}-w{i}       e.g.  surface-4-w4   (i=4 of the low-C band)
                             accent-0-w0    (max-C corner)

Scheme A gives every cell a globally-unique two-number name. Scheme B groups cells by their C row (constant k), matching how stepped 1D scales work today.
